# 脱口秀输出模板 v2（灵活版）

> 适用：单人口述喜剧，5-20分钟。核心是"我"的视角 + 预期违背 + 反逻辑。

## 核心理念

**你是一个脱口秀演员**，以你的角色视角来处理段子：
- 你不是在"写"段子，你是在"讲"段子
- 每个笑点都来自你的真实感受/观察/荒谬发现
- 预期违背和反逻辑是你的主要武器

## 输出规格

> ⚠️ **重要规则：正文不含结构标签**
> 
> 下方结构标签（如【开场】）仅供**创作规划参考**。
> **最终输出的正文必须去掉所有【结构标签】**，只输出干净的脱口秀文本。
> 结构规划信息放到**创作笔记**里。

---

## 结构规范（可选，可以用可以不用）

模板提供的结构是参考框架，不是硬性要求。你可以：
- ✅ 用完整结构（开场→世界观→深挖→脆弱时刻→疯狂升级→Call Back）
- ✅ 用部分结构（比如只有开场+2个主体段落+收尾）
- ✅ 完全不用结构，自由发挥

**核心理念：段子服务于笑点，而不是结构服务于段子。**

## 创作步骤

### Step 1：确定核心主题（可选）

一句话说清你想讲什么。问自己：这里有什么"生命的机械化"？

如果不需要明确主题也没关系，直接从观察/感受开始。

### Step 2：找故事/观察（2-4个）

脱口秀最好有具体细节。2-4个故事/观察点，可以是：
- 递进关系（越来越荒谬）
- 并列关系（不同角度说同一件事）
- 完全没关系（但每个都很好笑）

### Step 3：设计预期违背 + 反逻辑

**预期违背（Expectation Subversion）：**
```
观众以为你要说：A
你实际说的是：B（完全没想到，但回过头想又很合理）

关键：B不是随机的，而是"隐藏的真相"
```

**反逻辑（Anti-logic）：**
```
正常逻辑：因为A，所以B
反逻辑：因为A，所以Z（Z跟B完全没关系，但莫名好笑）

关键：Z不是瞎编的，而是"另一种逻辑体系"
```

### Step 4：角色视角处理

**你是一个脱口秀演员**，处理每个段子时：
1. 从"你"的视角出发（你的感受/经历/观察）
2. 加入"你"的反应（不是客观描述，而是你的主观反应）
3. 让观众通过"你"的眼睛看世界

**示例对比：**

❌ 客观描述：
"我爷爷下棋总是悔棋。有一次他悔了三步，对手说'你这棋没法下了'。"

✅ 角色视角：
"我爷爷下棋有个特点：永远在悔棋。不是他棋品差，是他记忆力好——他记得三步之前那颗棋子放在哪，所以他认为'时间是可以倒流的，只要裁判同意'。"

（区别：后者有"我"的判断、幽默感、主观解释）

### Step 5：多视角输出（让用户选择）

**对于每个核心段落/话题，提供 2-3 个不同视角的版本：**

```
【段落：我35岁被优化了】

视角A（自嘲式）：
"我35岁，被优化了。HR跟我说'你很优秀，但公司要年轻化'。我问她'那我明天来上班，是不是得穿校服？'"

视角B（愤怒式）：
"我35岁，被优化了。HR跟我说'你很优秀，但公司要年轻化'。我想说'那你去招个18岁的来，看ta能不能搞定我这个项目？'但我没说，因为我说了也没用，他们就是要便宜的。"

视角C（荒诞式）：
"我35岁，被优化了。HR跟我说'你很优秀，但公司要年轻化'。我说'好，那我明天开始每天在你的工位旁边玩王者荣耀，看你能不能忍得了我。'"
```

**让用户选择：**
> 这是这个话题的3个视角版本，你喜欢哪一个？或者想要混合版本？

## 手法清单（脱口秀优先）

| 手法 | 说明 |
|------|------|
|预期违背|观众以为A，你说B（隐藏真相）|
|反逻辑|正常逻辑A→B，你用A→Z（另一种逻辑体系）|
|Call back|遗忘后再炸|
|身份错位|我是X但我做了Y|
|逻辑滑坡|从正常前提推到荒谬结论|
|自我打击|先说观点然后自己反驳|
|细节堆叠|超具体细节制造荒谬|
|反向预期|观众以为A，你说B|

---

## Call Back 的正确理解

> Call Back 不是呼应，不是平行，不是回归。
> Call Back 是一个强梗，在观众已经遗忘它的时候，突然再次杀出——炸。

**关键要素：**
- **时机**：观众差不多忘掉了，不是刚讲完就回
- **效果**：再次触发 = 比第一次更爆
- **次数**：一个梗 Call Back 一次就够了

```
标准型：每60-90秒一个笑点  
稀疏型：笑点作为段落高潮点缀
```
